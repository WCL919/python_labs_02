# Лабораторная работа №1: Классы и инкапсуляция в Python

## Описание проекта
Эта работа создана для изучения классов и инкапсуляции в Python.
Мы реализовали класс `Student`, который хранит данные о студенте и считает его оценки.

## Структура файлов
- `model.py`: Содержит класс `Student` с логикой работы.
- `demo.py`: Показывает, как использовать класс `Student`.

## Что реализовано?
1.  **Класс `Student`**
    - Хранит имя студента, группу, оценку за экзамен и оценку за домашнее задание.
    - Все данные защищены, их нельзя изменить без проверки.

2.  **Проверка данных**
    - Оценки должны быть числами от 0 до 100.
    - Если ввести неправильные данные, программа покажет ошибку.

3.  **Методы класса**
    - `get_final_grade()`: Считает среднюю оценку.
    - `get_rating()`: Возвращает оценку словами (Отлично, Хорошо и т.д.).
    - `is_excellent()`: Проверяет, является ли студент отличником.

4.  **Вывод информации**
    - Метод `__str__` показывает удобную информацию о студенте.
    - Метод `__repr__` помогает отлаживать программу.
    - Метод `__eq__` сравнивает двух студентов.

## Как запустить?
1.  Сохраните файлы `model.py` и `demo.py` в одной папке.
2.  Запустите файл `demo.py`.
3.  Посмотрите результат в консоли: там будут данные студентов и тесты работы класса.

# Лабораторная работа №2: Класс-контейнер StudentCollection

## Описание проекта
Эта работа создана для изучения контейнерных классов в Python. Мы реализовали класс `StudentCollection`, который управляет коллекцией объектов `Student` из ЛР-1, поддерживает добавление, удаление, поиск, сортировку и другие операции.

## Структура файлов
*   `model.py`: Содержит класс `Student` с логикой работы (из ЛР-1).
*   `collection.py`: Содержит класс-контейнер `StudentCollection` для управления списком студентов.
*   `demo.py`: Показывает, как использовать класс `StudentCollection` и все его методы.

## Что реализовано?
### 1. Класс StudentCollection
*   Хранит список объектов `Student`.
*   Поддерживает добавление (`add()`) и удаление (`remove()`, `remove_at()`) студентов.
*   Проверяет тип данных: можно добавлять только объекты `Student`.
*   Запрещает добавление дубликатов студентов.

### 2. Методы поиска и доступа
*   `find_by_name()`, `find_by_group()`: Поиск студентов по имени и группе.
*   `__len__()`: Поддержка функции `len()` для получения количества студентов.
*   `__iter__()`: Поддержка цикла `for` для перебора коллекции.
*   `__getitem__()`: Доступ к студентам по индексу (`collection[0]`).

### 3. Сортировка и фильтрация
*   `sort_by_name()`: Сортировка студентов по имени.
*   `sort_by_final_grade()`: Сортировка по итоговому баллу.
*   `get_excellent_students()`: Фильтрация отличников (средний балл ≥ 90).

### 4. Обработка ошибок
*   Проверка корректности добавляемых объектов.
*   Обработка ошибок при удалении несуществующих студентов.
*   Защита от дубликатов и неверных типов данных.
