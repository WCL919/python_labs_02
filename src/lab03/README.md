# Лабораторная работа №3: Наследование и иерархия классов

## Цель работы
Освоить механизм наследования классов, научиться строить иерархию объектов, понять разницу между базовым и производным классом, освоить переопределение методов и реализовать полиморфизм.

## Структура файлов
*   `base.py`: Базовый класс `Student` (из ЛР-1)
*   `models.py`: Дочерние классы `BachelorStudent`, `MasterStudent`, `PhDStudent`
*   `demo.py`: Демонстрация работы всех методов и полиморфизма

## Описание реализованной иерархии
### Базовый класс: `Student`
Содержит общие данные для всех студентов: имя, группа, оценки за экзамен и домашнее задание. Реализует проверку данных, расчет среднего балла, определение рейтинга и методы для сравнения и вывода информации.

### Дочерние классы
1.  **`BachelorStudent` (Бакалавр)**
    *   Новые атрибуты: `course` (курс обучения), `scholarship` (наличие стипендии)
    *   Новый метод: `get_scholarship_amount()` — расчет размера стипендии
    *   Переопределены методы `__str__` и `get_rating()`

2.  **`MasterStudent` (Магистрант)**
    *   Новые атрибуты: `supervisor` (научный руководитель), `project_topic` (тема проекта)
    *   Новый метод: `get_project_details()` — получение деталей проекта
    *   Переопределены методы `__str__` и `get_rating()`

3.  **`PhDStudent` (Аспирант)**
    *   Новые атрибуты: `department` (кафедра), `dissertation_topic` (тема диссертации)
    *   Новый метод: `get_dissertation_progress()` — получение статуса диссертации
    *   Переопределены методы `__str__` и `get_rating()`

## Что реализовано?
### Требования на 3 балла
*   Использован класс `Student` из ЛР-1 как базовый
*   Создано 3 дочерних класса (выполнено требование минимум 2)
*   В каждом дочернем классе добавлено 2 новых атрибута и 1 новый метод
*   Использован `super()` для вызова конструктора базового класса, без дублирования кода
*   В `demo.py` демонстрировано создание объектов разных типов, вывод и вызов методов

### Требования на 4 балла
*   Выполнены все требования на 3 балла
*   Переопределены методы базового класса (`__str__`, `get_rating()`)
*   Использован `isinstance()` для проверки типов объектов
*   Реализовано полиморфное поведение: один и тот же метод работает по-разному в разных классах
*   Интеграция с коллекцией из ЛР-2: коллекция корректно хранит разные типы студентов

### Требования на 5 баллов
*   Выполнены все требования на 4 балла
*   Введен общий интерфейс через метод `get_student_type()` для всех классов
*   Реализован полиморфизм без анти-паттерна `if type == ...`
*   Добавлена фильтрация по типу студентов в коллекции
*   В `demo.py` демонстрировано единый стиль вызова методов для разных типов, фильтрация и полиморфизм

## Итог
Иерархия классов полностью реализована, выполнены все требования на 5 баллов. Реализовано наследование, переопределение методов, полиморфизм, интеграция с предыдущими лабораторными работами. Все методы работают корректно, код структурирован и не содержит дублирования.
